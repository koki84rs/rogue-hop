# Rogue Hop — ゲームデザインドキュメント

**バージョン**: 0.1 (企画段階)
**最終更新**: 2026-04-16

---

## 1. コンセプト

### 1.1 一言で

**タップ1つで遊べる、プールが延々と増え続けるローグライトジャンプアクション**

### 1.2 ターゲット

- スマホで隙間時間に遊ぶカジュアル層
- ローグライトの「シナジー発見」が好きな層
- The Binding of Isaac / Brotato / Slay the Spire 系の愛好家でスマホ操作を求める人

### 1.3 プラットフォーム

- **Phase 1**: スマホWeb（Phaser 3、URL共有で配布）
- **Phase 2**: iOS / Android アプリ（Capacitor でラップ）

---

## 2. 参考作品と設計原則

| 参考作品 | 本作に取り入れる要素 |
|---|---|
| **Brotato** | シンプル操作＋短時間ラン＋毎回違う体験 |
| **The Binding of Isaac** | 巨大アイテムプール、消去なし、ランダム発見の楽しみ |
| **Slay the Spire** | 3択でビルドが育つ緊張感 |
| **Dead Cells** | 永続アンロック（プール拡張型、再ロック不可） |
| **Nova Drift** | 幾何学＋ミニマル美術、形で種別がわかる敵 |

### 設計哲学（最重要）

**永続強化は「数値強化」ではなく「プール拡張」のみ**

- プレイヤーは毎ラン素の状態から始まる
- ラン達成で解放されるのは「新能力」「新敵」「新ボス」「新ステージ」のみ
- プレイヤー自体が強くなることはない（ステータス強化なし）
- アイテム消去・削除機能なし（Isaac 哲学）

→ この設計により「延々と新しい要素が増える」体感と、「シナジー発見の多様性」が無限に拡張する構造が成立する。

## 3. コアゲームプレイ

### 3.1 1ステージ（60秒）

```
[ 画面左寄り ]  自キャラ（画面中央から少し左、地上固定位置）
[ 画面右 ]     敵が画面右から左に向かって流入

60秒耐えきればクリア
ダメージを受けて HP が 0 になるとゲームオーバー
```

### 3.2 1ラン（約十数分）

```
ステージ1（60秒耐える）
  ↓ クリア
能力3択 → 1つ取得
  ↓
ステージ2（敵強化）
  ↓
能力3択
  ...
ステージ10
  ↓
能力3択（ボス前準備）
  ↓
ボス戦
  ↓
クリア（ラン成功） or ゲームオーバー
  ↓
メタ進行画面（解放ポイント付与、新要素アンロック）
  ↓
次のランへ
```

### 3.3 操作

| 入力 | 動作 |
|---|---|
| 短タップ（0.2秒未満） | ジャンプ |
| 長押し（0.2秒以上） | 滞空（エネルギー消費） |
| （それ以外） | なし |

**滞空の仕様**:

- 初期容量: 2秒分
- 地上接地中に回復（1秒地上 = 1秒分回復）
- 能力で容量UP、回復速度UP、滞空中攻撃などが追加される

## 4. 敵デザイン

### 4.1 Phase 1 敵ラインナップ（5種類）

| 形 | 名称 | 挙動 | 処理法 | 備考 |
|---|---|---|---|---|
| ■ 四角 | Walker | 地上を右→左に歩く | ジャンプで避け、または踏みつけて倒す | クリボー型。最も基本 |
| ▲ 三角（上向） | Spike | 地上を右→左、上部がトゲ | 踏めない。飛び越えるか落下攻撃で破壊 | トゲゾー型 |
| ● 円 | Shooter | 位置を保ち、定期的に弾を左に発射 | 弾をジャンプで避けつつ、接近して踏む | 砲台型 |
| ◆ 菱形 | Flyer | 中〜高空を水平飛行 | 高ジャンプで踏む、または滞空中攻撃 | 飛行型 |
| ▼ 逆三角 | Hopper | 地上で跳ねながら接近 | 敵の頂点タイミングを読んで踏む | ジャンプ型 |

**形＝挙動のヒント**という設計により、見た瞬間に処理方針が伝わる。

### 4.2 敵データ構造（enemies.json）

## 5. 能力システム

### 5.1 Phase 1 能力ラインナップ（10種類）

| カテゴリ | 能力名 | 効果 |
|---|---|---|
| 滞空拡張 | Extended Hover | 滞空容量 +2秒 |
| 滞空拡張 | Fast Recovery | 滞空回復速度 2倍 |
| 滞空拡張 | Air Shield | 滞空中、被弾無効（ただし滞空消費倍） |
| ジャンプ系 | Double Jump | 空中で1回追加ジャンプ可能 |
| ジャンプ系 | High Jump | ジャンプ高さ +40% |
| 攻撃系 | Dive Kick | 空中で下方向に急降下、トゲ敵も破壊 |
| 攻撃系 | Shockwave | 着地時に周囲の敵にダメージ |
| 攻撃系 | Bullet Reflect | 滞空中の接触で弾を反射 |
| 防御系 | Barrier | 1回の被弾を無効化（ラン中1回のみ、拾うたびに回復） |
| 特殊 | Slow Time | 滞空中、周囲の時間が0.5倍速 |

### 5.2 シナジー例（Isaac 型設計）

- `Dive Kick` × `Shockwave` → 急降下着地時の範囲ダメージ2段階
- `Double Jump` × `Bullet Reflect` → 高高度からの弾幕掃討
- `Extended Hover` × `Slow Time` → 滞空中の長時間スロー戦略
- `High Jump` × `Air Shield` → ほぼ無敵の滞空戦略（代わりに滞空消費大）

**重要**: 能力は単体でも機能するが、特定の組み合わせで真価を発揮する。この発見が周回動機。

### 5.3 能力データ構造（abilities.json）

```json
{
  "extended_hover": {
    "name": "Extended Hover",
    "description": "滞空容量 +2秒",
    "rarity": "common",
    "effect": { "type": "stat", "stat": "hoverCapacity", "value": 2 },
    "synergies": ["fast_recovery", "slow_time"]
  }
}
```

## 6. メタ進行

### 6.1 解放ポイント

- **獲得条件**:
  - ラン達成（ボス撃破）: 10pt
  - ステージ到達数に応じて: 到達ステージ数 × 1pt
  - 実績解除: 実績ごとに 3〜10pt

### 6.2 解放可能な要素

| カテゴリ | Phase 1 数 | Phase 2 以降追加予定 |
|---|---|---|
| 新能力 | 初期10 + 解放3 = 計13 | 段階的に+20, +50 |
| 新敵 | 初期5 + 解放2 = 計7 | +10, +20 |
| 新ボス | 初期1 + 解放0 | +2, +5 |
| 新ステージテーマ | 初期1 | +2 |

### 6.3 ロック解除UI

メタ進行画面で「解放ツリー」を表示。プレイヤーはポイントで自由に解放先を選べる。

- 解放した能力: 次ランから3択の中に出現し得る
- 解放した敵: 次ランから出現し得る（既存敵と混在）
- 再ロック・削除不可

## 7. 実績

### 7.1 方針

- 実績解放も解放ポイントを付与し、メタ進行に還元
- カテゴリ別に10個（Phase 1）、段階的に拡張

### 7.2 Phase 1 実績例

| カテゴリ | 実績例 |
|---|---|
| 探索系 | 初めてボスを倒す／全能力を1度ずつ取得 |
| 挑戦系 | ノーダメでステージ5到達／滞空だけで30秒／特定能力のみでクリア |
| 数字系 | 累計10ラン／累計敵撃破1000体 |
| シナジー系 | Dive Kick + Shockwave を同時所持／滞空能力3つ同時所持 |

## 8. 美術

### 8.1 ビジュアル方針

- **ミニマル＋ジオメトリック**（Nova Drift / Downwell 系）
- モノクロ（白背景＋黒オブジェクト）+ **アクセント1色（赤）**
- 赤は「ダメージ・危険・プレイヤーHP」を意味する

### 8.2 各要素の描画

| 要素 | 描画 |
|---|---|
| 自キャラ | 黒い三角（進行方向＝右を示す） |
| 地面 | 黒い水平線 |
| 背景 | 白（将来グラデーション可） |
| 敵 | 黒い幾何学図形（形で種別がわかる） |
| 弾 | 赤い小円 |
| プレイヤー被弾エフェクト | 赤いフラッシュ |
| UI数字 | 黒モノスペース |
| 滞空ゲージ | 画面上部の横バー（黒、減ると右から削れる） |

### 8.3 実装方針

Phaser 3 の `Graphics` API でコード内で直接図形を描画する。スプライト画像は使わない（アセット不要、色・形の調整が即反映される）。

## 10. 技術スタック

### 10.1 フロントエンド

| レイヤー | 技術 | 理由 |
|---|---|---|
| ゲームエンジン | Phaser 3 | 2D特化、スマホWeb対応、ドキュメント豊富 |
| 言語 | TypeScript | 型安全でバグが減る、IDE補完が効く |
| ビルドツール | Vite | 高速、Phaserテンプレートあり |
| パッケージ管理 | npm | Node.js 標準 |

### 10.2 配信・公開

| レイヤー | 技術 |
|---|---|
| ソース管理 | Git + GitHub |
| ホスティング | Netlify（GitHub連携で自動デプロイ） |
| 将来アプリ化 | Capacitor（iOS/Android） |

### 10.3 データ保存

- **Phase 1**: `localStorage`（ブラウザ内保存）
  - セーブデータ: 解放済み要素、実績進捗、累計数値
  - 容量上限: 5MB（十分）
- **Phase 2 以降**: Firebase Auth + Firestore（機種跨ぎ同期・リーダーボード）

## 11. プロジェクト構造

```
rogue-hop/
├── README.md
├── package.json
├── tsconfig.json
├── vite.config.ts
├── index.html
├── docs/
│   ├── GAME_DESIGN.md          ← このファイル
│   └── SETUP.md                ← 初心者向けセットアップ手順
├── public/
│   └── assets/
│       └── sounds/             ← SE ファイル
├── src/
│   ├── main.ts                 ← エントリポイント
│   ├── scenes/
│   │   ├── BootScene.ts        ← 起動・データロード
│   │   ├── MenuScene.ts        ← タイトル・メタ進行画面
│   │   ├── GameScene.ts        ← メインゲーム（60秒ウェーブ）
│   │   ├── AbilitySelectScene.ts  ← 能力3択画面
│   │   ├── BossScene.ts        ← ボス戦
│   │   └── GameOverScene.ts    ← ゲームオーバー・リザルト
│   ├── entities/
│   │   ├── Player.ts
│   │   ├── Enemy.ts
│   │   └── Projectile.ts
│   ├── systems/
│   │   ├── AbilityManager.ts   ← 能力の効果適用
│   │   ├── EnemySpawner.ts     ← 敵出現パターン管理
│   │   ├── MetaProgress.ts     ← 永続強化・解放管理
│   │   └── AchievementTracker.ts
│   └── data/
│       ├── enemies.json
│       ├── abilities.json
│       ├── achievements.json
│       ├── stages.json
│       └── bosses.json
```

**重要**: 敵・能力・実績・ステージの追加は `src/data/*.json` に1行追加するだけで済む。コード本体（src/systems/, src/entities/）には一切触らない。これが「延々と増える」を実現するデータ駆動アーキテクチャ。
